# 너구리의 모험 - 개발 작업 계획서

## 개요
본 문서는 "너구리의 모험" 게임 개발을 위한 단계별 작업 계획입니다.

---

## Phase 1: 프리 프로덕션 (Pre-Production)

### 1.1 기획 확정
- [ ] 게임 디자인 문서(GDD) 최종 확정
- [ ] 핵심 게임플레이 루프 정의
- [ ] 타겟 유저 페르소나 작성
- [ ] 경쟁작 분석 리포트 작성

### 1.2 아트 컨셉
- [ ] 캐릭터 컨셉 아트 (너구리 주인공)
- [ ] 적 캐릭터 컨셉 아트 (까마귀, 멧돼지, 뱀, 사냥꾼)
- [ ] 환경 컨셉 아트 (봄/여름/가을/겨울)
- [ ] UI/UX 와이어프레임 설계

### 1.3 기술 검토
- [ ] 게임 엔진 선정 (Unity / Unreal / Godot)
- [ ] 타겟 플랫폼별 기술 사양 정의
- [ ] 개발 환경 및 버전 관리 체계 구축
- [ ] 코딩 컨벤션 및 폴더 구조 정의

### 1.4 산출물
| 항목 | 담당 | 완료 기준 |
|------|------|----------|
| GDD 문서 | 기획 | 팀 리뷰 완료 |
| 컨셉 아트 10종 | 아트 | 승인 완료 |
| 기술 설계서 | 개발 | 문서화 완료 |

---

## Phase 2: 프로토타입 (Prototype)

### 2.1 핵심 메카닉 구현
- [ ] 캐릭터 이동 시스템 (좌우 이동, 점프, 이중 점프)
- [ ] 물리 엔진 적용 및 충돌 처리
- [ ] 꼬리 공격 시스템
- [ ] 변신 시스템 (나뭇잎 변신)

### 2.2 기본 시스템
- [ ] 체력(HP) 시스템
- [ ] 스태미나 시스템
- [ ] 아이템 수집 시스템 (도토리, 열매)
- [ ] 점수 시스템

### 2.3 테스트 스테이지
- [ ] 그레이박스 레벨 1개 제작
- [ ] 기본 장애물 배치
- [ ] 플레이 테스트 및 피드백 수집

### 2.4 산출물
| 항목 | 담당 | 완료 기준 |
|------|------|----------|
| 플레이 가능한 프로토타입 | 개발 | 핵심 루프 동작 |
| 테스트 리포트 | QA | 피드백 정리 |
| 메카닉 밸런스 초안 | 기획 | 수치 문서화 |

## Phase 3: 버티컬 슬라이스 (Vertical Slice)

### 3.1 월드 1 완성 - 봄의 숲
- [ ] 스테이지 1-1 ~ 1-5 레벨 디자인
- [ ] 봄 테마 환경 에셋 제작
- [ ] 배경 음악 및 효과음 적용
- [ ] 튜토리얼 시스템 구현

### 3.2 적 AI 구현
- [ ] 까마귀 AI (공중 이동, 급강하 공격)
- [ ] 멧돼지 AI (지상 돌진)
- [ ] 적 스폰 시스템

### 3.3 UI 구현
- [ ] 메인 메뉴 화면
- [ ] 인게임 HUD (체력, 점수, 스태미나)
- [ ] 일시정지 메뉴
- [ ] 스테이지 클리어 화면

### 3.4 산출물
| 항목 | 담당 | 완료 기준 |
|------|------|----------|
| 월드 1 완성본 | 전체 | 플레이 가능 |
| 에셋 리스트 | 아트 | 목록 정리 |
| QA 체크리스트 | QA | 버그 0건 |
